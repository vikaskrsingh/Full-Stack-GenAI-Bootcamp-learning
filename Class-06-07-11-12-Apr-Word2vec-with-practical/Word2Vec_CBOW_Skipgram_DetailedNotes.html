Vocabulary — The Universal Foundation
Required before any Word2Vec training can begin

Every text-to-number technique — OHE, BoW, TF-IDF, Word2Vec — starts by building a vocabulary. Word2Vec is no exception. The vocabulary is the complete set of unique words (tokens) found across all your training data.

📖 Vocabulary Definition

A vocabulary is a set of all unique words (tokens) present in the training data. Duplicate words are counted only once. The vocabulary determines:

→ In OHE/BoW/TF-IDF: the vector dimension (vocab size = vector length)

→ In Word2Vec: the size of the input/output layers of the neural network

✏️ Building Vocabulary — Worked Example from Class

Sentences:

S1: "I love AI"
S2: "I love coding"

Vocabulary = unique words only (case-sensitive!):

{I, love, AI, and, coding}

Note: "I" and "love" appear twice across sentences but appear only once in the vocabulary.

⚠️ Vocabulary Is Case-Sensitive

The instructor emphasized this clearly: "Sunny" (capital S) and "sunny" (lowercase s) are two different entries in the vocabulary, because vocabulary is case-sensitive.

Similarly, "Apple" and "apple" → both would be separate entries in the raw vocabulary. This is why preprocessing (converting all text to lowercase) is important before building a vocabulary — so "Sunny" and "sunny" are treated as the same word.

✏️ Case Sensitivity Example from Class
S1: "Sunny loves AI"    ← capital S
S2: "sunny loves coding"   ← lowercase s

Vocabulary = {Sunny, sunny, loves, AI, coding} → 5 words, not 4!

"Sunny" and "sunny" are treated as different words → different vector positions → different embeddings!

Best practice: convert all text to lowercase before building vocabulary to avoid this duplication.

02
Window Size — The Key Hyperparameter
How we convert raw text into training data for the neural network

To train the Word2Vec neural network, we need labeled data — X (input) and Y (output). But our text data is unstructured. The window size is the mechanism that converts our raw sentences into structured input-output pairs for supervised training.

— Sunny Savita
📖 Window Size Definition

Window size = how many neighbor words to consider on each side (left and right) of a target word.

Window size = 1 → consider 1 word to the left and 1 word to the right

Window size = 2 → consider 2 words to the left and 2 words to the right

Window size is a hyperparameter — no fixed correct value. You experiment to find the best one.

Why Is Window Size Needed?

  1. 1
    Word2Vec trains a neural network

    Neural networks need labeled data (X → input, Y → output) for supervised learning.

  2. 2
    Text is unstructured

    "I love deep learning" is not labeled. There's no X and Y column — just raw words.

  3. 3
    Window size converts text into (context, target) pairs

    By sliding a window across the sentence, we automatically generate training pairs without any human labeling. This is called self-supervised learning.

💡 Window Size Trade-offs — Critical Concept

Larger window size → more context around each word → richer understanding → BUT also more noise (irrelevant words get included)

Smaller window size → tighter, more specific context → less noise → BUT might miss important context

Typical values: 5–10 for most use cases. Google's paper used 5. This is a hyperparameter — you tune it through experimentation.

03
CBOW — Continuous Bag of Words
First of the two Word2Vec training architectures
📖 CBOW Definition

CBOW (Continuous Bag of Words) = given the context words, predict the target word.

Input (X): context words (surrounding words)

Output (Y): target word (the word in the center)

In CBOW, we give the model the surrounding words and ask it: "what word belongs in the middle?" That's how we create our training data automatically from raw text.

— Sunny Savita

Data Preparation for CBOW

Sentence: "Follow Sunny for Gen AI" | Window Size: 1

Follow, forSunnyLeft neighbor "Follow", right neighbor "for"
Sunny, GenforLeft "Sunny", right "Gen"

🧠 CBOW Training Goal

Train the neural network: given context words → predict the target word in the middle.

After training, the weight matrix that represents word relationships becomes the word embeddings.

04
Skip-gram — The Reverse of CBOW
Second Word2Vec architecture — predict context from target
📖 Skip-gram Definition

Skip-gram = given the target word, predict the context words around it.

Input (X): target word (the center word)

Output (Y): context words (surrounding words)

It's the UNO reverse of CBOW — simply swap input and output.

For Skip-gram, we play the UNO reverse. The target becomes the context, and the context becomes the target. That's the only difference between CBOW and Skip-gram at the data level.

— Sunny Savita

🎯 Key Interview Insight

The difference between CBOW and Skip-gram is only in how training data is prepared. The neural network architecture (1 hidden layer, same input/output layer size) is identical. Just the direction of prediction is reversed.

Google trained their official pre-trained model using Skip-gram because Skip-gram performs better with large, diverse datasets like Google News.

05
How the Neural Network Is Trained
Step-by-step: from raw text to a trained Word2Vec model
  1. 1
    Take raw text data

    Any corpus of text — a sentence, a book, a news archive. Word2Vec learns from the patterns in this data.

  2. 2
    Build vocabulary

    Extract all unique words. Vocabulary size = V. This determines the size of input and output layers.

  3. 3
    Create one-hot encoded vectors for each vocabulary word

    Each word becomes a V-dimensional binary vector with 1 at its position and 0 everywhere else. These are the initial representations.

  4. 4
    Prepare training pairs using window size

    Using CBOW or Skip-gram approach, generate (context, target) or (target, context) pairs from the text.

  5. 5
    Feed data into neural network and train

    Input = one-hot vectors of context/target words. Run forward propagation → compute loss → run backward propagation → update weights. Repeat for many epochs.

  6. 6
    Extract the embedding

    After training, the best-fit weight matrix of the hidden layer IS the embedding matrix. Each row = the embedding vector for one vocabulary word.

📐 Matrix Sizes — Why?

W₁ (Input → Hidden): 5 × 3 → 5 vocab words × 3 hidden neurons → 15 learnable weights

W₂ (Hidden → Output): 3 × 5 → 3 hidden neurons × 5 output positions → 15 learnable weights

Output layer: 5 neurons (one per vocabulary word). After Softmax, gives probability distribution over all words. Highest probability = predicted word.

In Google's real model: vocab size V = millions, hidden layer = 300. So W₁ = V × 300 and W₂ = 300 × V.

06
Where Exactly Do Embeddings Come From?
The most important conceptual point of the entire session

The embedding is the output of the hidden layer after training. The best possible weights, after all the forward and backward propagations — that weight matrix IS the embedding. Each word's row in that matrix is its embedding vector.

— Sunny Savita
  1. 1
    Initialize weights randomly

    W₁ and W₂ start with random values between some range (e.g., -1 to 1). At this point the model knows nothing.

  2. 2
    Forward Propagation (FP)

    Pass input (one-hot vector of context word) → multiply by W₁ → get hidden layer output → multiply by W₂ → apply Softmax → get predicted probability for each vocabulary word.

  3. 3
    Calculate Loss

    Compare predicted output (probability distribution) with actual target (one-hot vector of target word). The difference = loss. Initially loss is high because weights are random.

  4. 4
    Backward Propagation (BP) — Gradient Descent

    Use the loss to calculate how much to adjust each weight. Update W₁ and W₂ in the direction that reduces loss. This is gradient descent.

  5. 5
    Repeat for many epochs

    One epoch = one full pass through all training pairs (FP + BP). Run hundreds or thousands of epochs. With each epoch, weights improve → loss decreases → model understands context better.

  6. 6
    Extract embeddings from W₁

    Once training is done, W₁ (the input-to-hidden weight matrix) is the embedding matrix. Row i of W₁ = the embedding for word i. The hidden layer output for any input word = that word's embedding vector.

🔬 Why Is the Hidden Layer Output = Embedding?

During training, the model learns to predict target words from context words. To do this correctly, it must understand which words appear in similar contexts.

Words that appear in similar contexts (e.g., "cat" and "dog" both appear near "pet", "feed", "play") will develop similar hidden layer representations — similar embedding vectors.

So the hidden layer, after training, encodes meaning through context. The output of the hidden layer for any word = a vector that captures that word's semantic meaning.

That's why hidden layer output = embedding. It's not designed explicitly — it emerges from the training process.

🔑 No Activation Function in the Hidden Layer

A student asked: "Why no activation function in Word2Vec?" Sunny's answer: Because the goal here is not classification or non-linear decision making — we just want a linear transformation of the input to produce a dense vector representation. Adding an activation function would distort the embedding values. The output layer uses Softmax for probability, but the hidden layer is intentionally linear.

Embedding Dimension = Hidden Layer Size
This is the critical formula that connects architecture to output
📐 The Golden Rule

OHE / BoW / TF-IDF vector dimension = Vocabulary size (fixed by data)

Word2Vec embedding dimension = Number of neurons in the hidden layer (hyperparameter — you choose it!)

ModelHidden Layer SizeEmbedding Dimension
Our classroom example3 neurons3-dimensional vectors
Google's pre-trained model300 neurons300-dimensional vectors
Custom model (you decide)Any numberSame as hidden layer
LLMs (e.g., GPT-4)ThousandsThousands of dimensions
💡 Higher Dimension = More Capacity to Capture Meaning

More neurons in the hidden layer → higher-dimensional vectors → more features can be captured → model can represent more nuanced word relationships.

But: more neurons = more computation = longer training time. 300 dimensions is the widely accepted sweet spot for most NLP tasks.

Like LLMs: a 1B parameter model vs 70B parameter model — more parameters = more capacity, but also much more compute required.

Python — Semantic Similarity Operations
# Find words most similar to "man"

# Output: [('woman', 0.76), ('boy', 0.73), ('teenager', 0.68), ...]
# Trained on Google News → criminal news may bias "man" toward suspect-related words

# Find words most similar to "cricket"
print(model.most_similar("cricket"))
# Output: [('cricketing', 0.82), ('cricketers', 0.80), ('Test Cricket', 0.79), ...]

# Check similarity between two words (uses cosine similarity internally)
print(model.similarity("man", "woman"))  # ~0.76 — 76% similar
print(model.similarity("man", "man"))    # 1.0 — identical words = 100%

print(model.similarity("Python", "Java"))# very low — treated as unrelated

# Case sensitivity demonstration
print(model.similarity("Sunny", "Sunny"))  # 1.0 — identical
print(model.similarity("Sunny", "sunny"))  # ~0.43 — DIFFERENT! Case sensitive!

# Find the odd one out
print(model.doesnt_match(["PHP", "Java", "dog", "C++"]))  # → 'dog'

Part 4 — The Famous King − Man + Woman = Queen

Python — Vector Arithmetic
# The classic Word2Vec demonstration
# king - man + woman should give something close to queen
result_vector = model["king"] - model["man"] + model["woman"]

# Find the most similar word to that computed vector
print(model.most_similar([result_vector]))
# Output: [('queen', 0.71), ('princess', 0.65), ...]  ← Queen appears near top!

# This proves Word2Vec has learned that:
# king and queen have the same relationship as man and woman
# Meaning is encoded in the vector arithmetic itself!

Average Word2Vec — Sentence-Level Embedding
How to convert an entire sentence into a single vector using Word2Vec
📖 Problem: Word2Vec Only Works on Single Words

The Word2Vec model converts one word at a time into a 300-dimensional vector. But in real applications, we usually need to represent entire sentences or paragraphs as single vectors (e.g., for RAG retrieval, document similarity).

The solution: Average Word2Vec — get a vector for each word in the sentence, then take the average across all words to produce one sentence vector.

How Average Word2Vec Works

  1. 1
    Tokenize the sentence

    "I love machine learning" → ["I", "love", "machine", "learning"]

  2. 2
    Get embedding for each word

    model["I"] → 300-dim vector, model["love"] → 300-dim vector, etc.

  3. 3
    Stack all word vectors into a matrix

    4 words × 300 dimensions = 4 × 300 matrix

  4. 4
    Take the column-wise average

    Average across all word vectors → one 300-dimensional sentence vector

Python — Average Word2Vec for Sentence Embedding
import numpy as np

# The sentence to embed
sentence = "I love machine learning"

# Step 1: Tokenize (lowercase + split)
words = sentence.lower().split()
# → ['i', 'love', 'machine', 'learning']

# Step 2: Get vector for each word
word_vectors = []
for word in words:
    try:
        word_vectors.append(model[word])
    except KeyError:
        pass  # skip OOV words

# Step 3 & 4: Stack vectors + take column-wise average
sentence_vector = np.mean(word_vectors, axis=0)

# Result: one 300-dimensional vector for the entire sentence
print(sentence_vector.shape)   # (300,)
print(sentence_vector)          # [0.123, -0.456, ...] — 300 floats
⚠️ Average Word2Vec Is Old Technology

Average Word2Vec was popular around 2015–2016. It works but has a key limitation: it ignores word order. "I love AI" and "AI love I" would produce the same sentence vector because averaging destroys sequence information.

Modern SOTA models (Sentence-BERT, OpenAI embeddings, Gemini embeddings) produce context-aware sentence embeddings that account for order and meaning. Those are what we use today in RAG systems.

Word2Vec — Pros & Cons
What made it revolutionary, and why we moved on
✅ Advantages
  • Efficient & fast training — shallow network (only 1 hidden layer) → quick to train
  • Semantic understanding — similar words get similar vectors (King ≈ Queen, car ≈ vehicle)
  • Vector arithmetic works — King − Man + Woman ≈ Queen. Relationships encoded in vectors
  • Pre-trained model available — Google's model can be used immediately, no training needed
  • General-purpose embeddings — works for NLP pipelines, ML modeling, clustering
  • Still used inside Transformers — initial token embedding layer in Transformers uses Word2Vec-style embedding
❌ Disadvantages
  • Static embeddings — "bank" in "river bank" and "bank account" get the SAME vector. Context is not considered
  • Out-of-Vocabulary (OOV) — words not in training data can't be embedded (e.g., "ChatGPT" fails on 2013 model)
  • Cannot handle sub-words — "running", "runner", "runs" are unrelated; no morphological understanding
  • No sentence-level context — Average Word2Vec loses word order
  • Case sensitivity issue — "Sunny" and "sunny" are different words
  • Replaced by transformers — GPT, BERT, and modern LLMs produce dynamic, context-aware embeddings
🔗 Word2Vec Still Lives Inside Transformers

When the instructor showed the "Attention is All You Need" paper, he pointed to the Input Embedding block at the bottom of the diagram. That block converts tokens into embeddings using a Word2Vec-style mechanism before they enter the multi-head attention layers.

So Word2Vec isn't completely obsolete — it's the foundation layer. The Transformer then refines these initial embeddings through self-attention to produce dynamic, context-aware representations.

🔴 The Static Embedding Problem — Explained

Consider the word "bank":

→ "I went to the river bank" → "bank" = water's edge

→ "I deposited money at the bank" → "bank" = financial institution

Word2Vec gives BOTH usages the exact same 300-dimensional vector — because it was trained on the whole corpus and the vector averages all contexts. It cannot change the embedding based on the sentence it's currently in.

This is why BERT (2018) was revolutionary — it introduced contextual embeddings: the same word "bank" gets a different vector depending on what sentence it's in.

Interview Preparation
All the questions from real GenAI interviews — answers included

🎯 Word2Vec Interview Q&A

Q1: What is Word2Vec?
Word2Vec is a neural network-based technique introduced by Google in 2013 for representing words as dense, fixed-dimensional vectors (embeddings). Unlike OHE/BoW/TF-IDF which use counting, Word2Vec trains a shallow neural network to predict words from context (or context from words), and the learned hidden layer weights become the word embeddings. Similar words end up with similar vectors — "King" and "Queen" are close, "car" and "vehicle" are close.
Q2: What is the difference between CBOW and Skip-gram?
Both are architectures inside Word2Vec — the neural network structure is the same (one hidden layer). The difference is only in data preparation. CBOW: given context words → predict the target word. Skip-gram: given the target word → predict context words. Skip-gram performs better with large datasets (like Google News) and handles rare words better. Google's official pre-trained model uses Skip-gram.
Q3: What is window size in Word2Vec?
Window size defines how many neighbor words to consider on each side of a target word when creating training pairs. Window size = 1 means 1 word left and 1 word right. Window size = 2 means 2 on each side. It's a hyperparameter — larger window = more context but also more noise. You find the optimal value through experimentation.
Q4: What is the embedding dimension in Word2Vec, and how is it determined?
The embedding dimension equals the number of neurons in the hidden layer. It's a hyperparameter — you choose it. Google's pre-trained model uses 300 dimensions (found through experimentation). In your custom model, you can use any value: 50, 100, 300, 500. More dimensions = more capacity to capture meaning, but more compute. The vector for every word will have this many numbers.
Q5: Where does the embedding come from in the Word2Vec neural network?
The embedding is the output of the hidden layer after training completes. During training, the weight matrix W₁ (connecting input to hidden layer) gets optimized via forward and backward propagation to minimize prediction loss. Once training is done, the rows of W₁ are the embeddings — each row corresponds to one vocabulary word's embedding vector.
Q6: What is the biggest limitation of Word2Vec?
Static embeddings — Word2Vec gives each word one fixed vector regardless of context. "Bank" in "river bank" and "bank account" get the same vector. This is a major limitation. Modern transformer-based models like BERT create contextual embeddings — the same word gets a different vector depending on the surrounding sentence. This was the key innovation that made BERT and GPT transformative.
Q7: Is Word2Vec still used today?
Yes — indirectly. The initial token embedding layer inside every Transformer architecture uses a Word2Vec-style lookup: tokens are mapped to dense vectors before entering the self-attention mechanism. After that, the Transformer's attention layers refine these embeddings into context-aware representations. So Word2Vec is the foundation, and attention mechanisms build on top of it.
Q8: What is Average Word2Vec and when was it used?
Average Word2Vec is a technique for creating sentence-level embeddings using Word2Vec. Get the embedding for each word in the sentence, then take the element-wise average of all word vectors to produce a single sentence vector. Popular around 2015–2016. It's simple but loses word order. Replaced by sentence transformers (Sentence-BERT, etc.) which produce better sentence embeddings by considering the full sentence context.
